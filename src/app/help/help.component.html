Aim

The aim of this exercise is to create a login page so that users can authenticate and view a dashboard.

Objectives

Primary objectives

  • Create a login form that has an email input, a password input and a submit button. Use the LoginComponent for this.
  • On successful login the user is redirected to the dashboard page.
  • The /dashboard route must only be accesible to authenticated users.

Secondary objectives

  • The login page should be styled similar to the image provided.
  • The email field should have a required validator.
  • The email field should have a valid email format validator.
  • The password field should have a required validator.
  • If an incorrect email and password combination is entered an error message should be displayed on submit.
  • If am logged in the /login route should redirect to the dashboard.
  • Dispay the user's details on the dashboard in any way you like.

Hints

  • Focus on completing the primary objectives first before moving onto the secondary ones.
  • Use the AuthenticationService to hold the state of your logged in user. You will need to implement the login() method here.
  • An AuthGuard has already been set on the /dashboard route, but you will need to implement the canActive() method.
  • The ApiService provides a fake endpoint to authenticate a user, so you don't need to worry about cookies or storing tokens in local storage or anything.
  • The login() method in ApiService will return a User if the email and password match those stored in usersDb. So you can test your login form using the email and password of the user defined there.
